2. LogiPat 手冊                          logiPat-manual   logiPat-man 

         logiPat-arg   logiPat-input   logiPat-pattern   logiPat-operators 
        布爾邏輯模式的組成是

                        operators  ! = 非
                                   | = 邏輯或
                                   & = 邏輯與
                        grouping   ( ... )
                        patterns   "pattern"

        :LogiPat {布爾邏輯模式}                          :LogiPat 
                :LogiPat 是一個接受布爾邏輯參數的命令 ( logiPat-arg )。

        :LP {布爾邏輯模式}                               :LP 
                :LP 是 :LogiPat 的簡寫命令版本

        :LPE {布爾邏輯模式}                              :LPE 
                不執行搜索，但進行布爾邏輯模式到正規表達式的轉換，並在屏幕上回
                顯。

        :LogiPatFlags {搜索標誌位}                       LogiPat-flags 
                :LogiPatFlags {search flags}
                LogiPat 使用  search()  命令。:LogiPatFlags 命令可用來指定傳遞
                給 search() 調用的標誌位。

        :LPF {搜索標誌位}                                :LPF 
                :LPF 是 :LogiPatFlags 的簡寫版本。

        :let pat=LogiPat({布爾邏輯模式})                 LogiPat() 
                如果直接調用 LogiPat()，不執行搜索，但進行布爾邏輯模式到正規表
                達式的轉換並返回之。

        要在模式裡指定 " 而不是用作模式的定界，給雙引號加倍。



3. LogiPat 示例                                          logiPat-examples 

        LogiPat 接受布爾邏輯參數，產生正規表達式來實現選擇。下面提供一系列的例
        子 :

        :LogiPat "abc"
                會搜索包含字符串 :abc: 的行

        :LogiPat "ab""cd"
                會搜索包含字符串 :ab"c: 的行

        :LogiPat !"abc"
                會搜索不包含字符串 :abc: 的行

        :LogiPat "abc"|"def"
                會搜索包含字符串 :abc: 或 :def: 的行

        :LogiPat !("abc"|"def")
                會搜索既不包含字符串 :abc: 也不包含 :def: 的行

        :LogiPat "abc"&"def"
                會搜索同時包含字符串 :abc: 和 :def: 的行

        :let pat= LogiPat('!"abc"')
                會返回正規表達式，可用來匹配所有不包含 :abc: 的行。雙倍的引號
                是必要的，它可以把正常的匹配傳遞給 LogiPat，並把這種模式和布爾
                邏輯的操作符區別開來。



4. 警告                                                  logiPat-caveat 

         "not" 操作符可能比較脆弱；也就是，它並不總是能和 & (邏輯與) 和 | (邏
         輯或) 操作符合作得很好。請先試試你的模式，可能的話用 :set hls，以確保
         匹配符合你的期望。
